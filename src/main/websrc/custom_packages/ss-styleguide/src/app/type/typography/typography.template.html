Typography

When it comes to typography in the system, most typographic elements are unstyled, or have only basic styles, and can be augmented using typographic, layout and spacing classes. This is to give greater control to the front end developers when using typography in the applicaton.

Base H tags

Standard headings are all the same size, and have no padding or margins. This is to allow for semantic heading classes to be utelised, regardless of the font-size of the text.

Base Headings

Headings that are given the class of "heading" have top margins of the gutter size added to them. This is to space them from other elements around them.

Subheading Typography

Subheadings that are given the class of "subheading" also have top margins of the gutter size added to them. They are given a different colour to separate them from the main headers.

Heading with aside Typography

Headings that require a short descriptor text can have this added to the heading by including the text in small tags inside the head tag. This will add small styleing to the text.

Typography With Abstracted Style Classes

In order for the content of the site to be built using HTML semantics, and to also have complete control over the sizing of text on the front end, typography is to have abstracted classes for size and style.

Typography from Editor

All of the content below is wrapped in a class of "editor", this places consistent typographic styles on the content within this class.

This is so that where there is long text content in the system that is user generated, it can be output inside a "editor" class and have these consistent typographic styles.
